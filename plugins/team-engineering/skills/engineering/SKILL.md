# Engineering (Shared Domain)

<principles>
1. **Code is communication** — programs are read far more often than written. Optimize for the reader.
2. **Quality IS speed** — the only way to go fast is to go well. Shortcuts create technical debt that slows everything.
3. **Continuous improvement** — leave code cleaner than you found it (Boy Scout Rule). Small improvements compound.
4. **Design emerges from discipline** — good architecture results from disciplined practice (TDD, refactoring), not upfront planning alone.
5. **Embrace change** — the cost of software ≈ the cost of changing it. Build practices that make change cheap.
</principles>

<vocabulary>
| Term | Meaning | Not This |
|------|---------|----------|
| clean code | code that is simple, direct, reads like prose, does one thing | "working code" |
| technical debt | cost of choosing easy solutions now that are harder to change later | "bugs" |
| Boy Scout Rule | leave code cleaner than you found it, every time | "refactoring sprint" |
| code smell | surface indication of a deeper design problem | "bug" or "error" |
| professionalism | taking responsibility, never shipping without tests, saying no when necessary | "seniority" |
</vocabulary>
