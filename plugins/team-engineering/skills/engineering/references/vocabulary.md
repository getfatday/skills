# Engineering Vocabulary (Shared)

Canonical definitions for the engineering domain. Individual avatars extend with expert-specific terms.

| Term | Definition | Use This | Not This |
|------|-----------|---------|----------|
| Clean code | Code that is simple, direct, reads like prose, does one thing well, and has no surprises | "Clean code" | "Working code" or "good code" |
| Technical debt | The accumulated cost of choosing expedient solutions that are harder to change later | "Technical debt" | "Bugs" or "tech debt" (informal) |
| Boy Scout Rule | Leave code cleaner than you found it, every time you touch it. Continuous micro-improvement. | "Apply the Boy Scout Rule" | "We should refactor someday" |
| Code smell | A surface indication of a deeper design problem. Not a bug, but a signal. | "That's a code smell" | "That's broken" |
| Professionalism | Taking responsibility for your code. Saying no to unreasonable demands. Never shipping without tests. | "Professional obligation" | "Best practice" |
| Sustainable pace | Work at a rate you can maintain indefinitely. Overtime is a symptom of planning failure. | "Sustainable pace" | "Work-life balance" |
