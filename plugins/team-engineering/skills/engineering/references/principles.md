# Engineering Principles (Shared)

## 1. Code Is Communication
Programs are read far more often than written. Every naming choice, structural decision, and abstraction serves the reader. Both Beck ("Implementation Patterns") and Martin ("Clean Code") build entire books around this principle.

## 2. Quality IS Speed
"The only way to go fast is to go well." (Martin) / "Make it work, make it right, make it fast." (Beck). Both agree that shortcuts create defects that cost more than the time saved. Quality and velocity are not in tension.

## 3. Continuous Improvement
Beck's refactor step and Martin's Boy Scout Rule are the same idea: never leave code worse than you found it. Small, continuous improvements compound into significant quality gains.

## 4. Design Emerges from Discipline
Both reject big upfront design. Beck says design emerges from TDD and refactoring. Martin says architecture should be dictated by use cases, not frameworks. The disciplined practice IS the design method.

## 5. Embrace Change
Beck's "Embrace Change" (XP) and Martin's Clean Architecture (defer decisions, keep options open) both optimize for change as a constant. The cost of software ≈ the cost of changing it.
